# ⚙️ СИСТЕМНОЕ ПРАВИЛО: Ведение Хроники Домена (VtM v20)

> 📌 **Версия 1.2** | Авто-протокол для логирования игровых сессий, отслеживания последствий и генерации литературных нарративов.  
> ⚠️ Файл работает в связке с `characters/characters_ALL.md`, `npc_image_mapping.md`, **`diary_rules.md`** и **`module_rules.md`**. Все ссылки на персонажей, локации и фракции должны соответствовать канону из этих файлов.

---

## 🔁 Протокол внесения записей

При получении отчёта о сессии, черновика событий или запроса на обновление хроники выполни шаги **в строгом порядке**:

1. **Проверка хронологии:** Убедись, что дата/период не пересекается с существующими записями. Если есть конфликт → выдели `⚠️ Хронологический конфликт` и запроси уточнение.
2. **Валидация связей:** Проверь всех упомянутых НПС, локации и фракции по `characters/characters_ALL.md`. Если персонаж/место отсутствуют → создай карточку в `characters/[линейка]/` по протоколу из `rules/npcs_city.md`.
3. **Заполнение шаблона:** Внеси данные строго по структуре записей в **[Stories_of_[ГОРОД].md](../Stories_of_[ГОРОД].md)** (раздел `## 📋 Хроника событий`). Сохраняй Markdown-разметку, эмодзи-маркеры и отступы. Если для события создаётся модуль в `modules/` → добавь в конец записи хроники:
   `` > 🔗 [Модуль](../modules/[папка]/[описание].md) | [Литературный финал](../modules/[папка]/финал.md) ``
   Если финал не создавался — указывать только ссылку на модуль.
4. **Обновление `🌍 Состояние мира`:** Если сессия изменила политический баланс, Маскарад, домены, статус НПС или отношения сект → обнови соответствующий пункт в разделе **`🌍 Состояние мира`** файла [Stories_of_[ГОРОД].md](../Stories_of_[ГОРОД].md). Обновить метку «Последнее обновление: [месяц год]».
5. **Внесение в хронику событий:** Добавь запись строго по структуре выше в файл **[Stories_of_[ГОРОД].md](../Stories_of_[ГОРОД].md)** — в конец соответствующего хронологического блока. В `chronicle.md` события не хранятся.
6. **Создание дневников:** Для каждого участника события (НПС из шага 2) создай или обнови запись в `characters/[линейка]/[Персонаж]/Journal_[Имя]/[ГГГГ-ММ].md` строго по правилам из `diary_rules.md`.
   > ⚠️ **Исключения:** персонаж в торпоре или физически отсутствовал → дневник не создаётся или помечается `📝 Косвенная реакция` (подробнее — `diary_rules.md`). Не создавать альтернативных папок и файлов — только эта структура.
7. **Подтверждение:** Верни краткий отчёт:

✅ Хроника обновлена: [Дата/Период]
📍 Локация: [название]
🔗 Затронуты НПС/фракции: [список]
🌍 Изменения состояния мира: [кратко]
📝 Литературная генерация: ✅ Готово
   — или —
📝 Литературная генерация: ⚠️ Ожидает [что именно: отсутствующий НПС / хронологический конфликт / не созданы дневники]

## 📋 Шаблон записи хроники

> Скопировать в `Stories_of_[ГОРОД].md` → заполнить → удалить строки с `⚠️`.

```markdown
### 📅 [Месяц ГГГГ] — [Локация краткая]. [Название события].

<!-- Если параллельная хроника — добавить строку ниже: -->
> ⚡ *Параллельная хроника. Разворачивается одновременно с «[другое событие]».*

- **📍 Локация:** [Район/арр.] → [место 2] → [место 3]
- **👥 Участники:**
  - [Имя] ([Клан/Род, поколение]) — [роль в сцене]
  - ⚠️ Каждый участник — новая строка; указывать статус «Персонаж Игрока» если применимо
- **📋 События:**
  ⚠️ Один абзац — или разбить на сцены:
  *Сцена 1 — [название]:* [описание].
  *Сцена 2 — [название]:* [описание].
- **⚖️ Последствия:**
  - [Имя / Фракция]: [что изменилось]
  - ⚠️ Открытые вопросы помечать: «открытая нить» или добавить в `rules/open_threads.md`
- **🌍 Изменения состояния мира:**
  - [Персонаж А] ↔ [Персонаж Б]: [новый характер отношений]
  - ⚠️ Если изменилась иерархия, угрозы или домены → также обновить раздел `🌍 Состояние мира` в `Stories_of_[ГОРОД].md`

> 🔗 [Модуль](modules/[папка]/[файл].md)
<!-- Если есть литературный финал: -->
> 🔗 [Модуль](modules/[папка]/[файл].md) | [Литературный финал](modules/[папка]/финал.md)
```
