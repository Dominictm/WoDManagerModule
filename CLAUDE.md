## Обязательные элементы каждого сценария

### 1. Структура

Завязка → 3–5 сцен → кульминация → последствия.

### 2. Городская специфика

В каждом сценарии указывать:

- Конкретные районы с реальной географией.
- Как сцена взаимодействует с городом смертных — прохожие, полиция, СМИ.
- Языковой нюанс — если Котери из иностранцев: языковой барьер, местные обычаи.
- Status и Etiquette — нарушение этикета на Элизиуме = серьёзный удар по репутации.

### 3. Блок NPC — местная форма

К стандартному блоку добавлять:

- Год Embrace и место (Embrace в ключевую историческую эпоху = особый политический вес).
- Положение в местной иерархии (Примоген, Гарпия, Шериф, Хранитель Элизиума, анцилла и т.д.).
- Связи с историей домена — участвовал ли в ключевых событиях, был ли в эмиграции.

### 4. Тематические лейтмотивы

> [ЗАПОЛНИТЬ] — выбери 4–6 тем, актуальных для твоего домена:

**Универсальные темы VtM:**
- **Старые обиды** — предательства прошлых веков, живые (вечно живые) свидетели
- **Власть и коррупция** — Камарилья как инструмент контроля, не защиты
- **Маскарад под давлением** — смартфоны, соцсети, видеонаблюдение
- **Чудовище внутри** — Зверь, потеря человечности, границы морали
- **Анархи vs. Установленный порядок** — идеологический конфликт, не просто банды
- **Угроза извне** — Шабаш / независимые / Инквизиция

## Чего избегать

- Не выдумывать имена канонических старейшин (Князь, Примогены, известные Methuselah). Если канон неоднозначен — предлагать варианты, помечать «не подтверждено».
- Не смешивать с другими доменами — у каждого города уникальный политический стиль.
- Не превращать Анархов в простых бандитов — они часто идеологизированы.
- Не использовать «воду» — каждое описание должно нести функцию (зацепка, угроза, информация).
- Не игнорировать языковой / культурный контекст локального домена.

> [ЗАПОЛНИТЬ] — добавь правила, специфичные для твоего домена:

> - Договорённости с игроками

## Персонажи проекта

Карточки персонажей хранятся в папке `characters/`, рассортированные по линейкам WoD:
- `characters/vampires/` — вампиры (по одному файлу на персонажа)
- `characters/fairies/` — феи / ченджлинги
- `characters/mortals/` — смертные
- `characters/werewolves/`, `mages/`, `hunters/` — пусты, готовы к заполнению

Сводный справочник всех персонажей — `characters/characters_ALL.md` (краткая информация + ссылки на арт + поле Лист).

Протокол добавления новых НПС — `rules/npcs_city.md`. Таблица привязки визуальных референсов — `rules/npc_image_mapping.md`. Правила промтов для изображений — `rules/portret.md`. Протокол ведения хроники — `rules/chronicle_paris.md`. Правила дневников — `rules/diary_rules.md`. Правила модулей — `rules/module_rules.md`. Открытые сюжетные нити — `rules/open_threads.md`. Правила V20-листов персонажей — `rules/character_sheet_v20.md`. Карта фракций домена — `factions.md`. Слухи Элизиума (d20) — `rumors_elysium.md`. Слухи Грёзы (d20, если есть феи) — `rumors_dreaming.md`. Хронология WoD домена — `rules/timeline.md`.

**Карточки локаций** хранятся в `locations/`, рассортированные по районам (`[Район]/`). Локации вне города — `locations/Другие/[модуль]/`. Правила создания — `rules/portret.md`.

**Мастер-чеклист** (новый персонаж / новый модуль / переименование) — `rules/CHECKLIST.md`.

**Утилиты** (`tools/`):
- `tools/validate_links.ps1` — проверка всех внутренних ссылок
- `tools/new_module.ps1` — создаёт папку модуля с файлами-заготовками
- `tools/new_npc.ps1 -Name "Имя" -Type vampire` — создаёт карточку НПС
- `tools/promote_npc.ps1 -ModuleName "..." -NpcName "Имя"` — продвигает НПС из модуля в канон
- `tools/new_location.ps1 -Name "..." -District "Район" -Zone neutral` — карточка локации
- `tools/add_diary.ps1 -CharacterName "Имя" -Period "ГГГГ-ММ"` — дневниковая запись
- `tools/status.ps1` — дашборд статусов персонажей
- `tools/search.ps1 "запрос"` — поиск по всем .md файлам проекта

При упоминании любого персонажа сверяться с его файлом в `characters/`. Новых NPC добавлять строго по протоколу из `rules/npcs_city.md`.
