> 📌 **Версия 1.0** | Протокол создания и хранения файлов модулей.  
> 🔗 Работает в связке с `chronicle_paris.md`, `diary_rules.md` и `characters/characters_ALL.md`.

---

## 📂 СТРУКТУРА — ЕДИНСТВЕННО ВЕРНЫЙ ФОРМАТ

```
modules/
└── [месяц_ГГГГ_описание]/     ← строчными буквами, через подчёркивание
    ├── [описание].md           ← всегда; лог сессии; имя = описание папки
    ├── сценарий.md             ← всегда; инструмент Мастера — до сессии
    ├── финал.md                ← только если сессия разыграна и финал написан
    ├── нпс.md                  ← всегда; список всех НПС (каноничных и модульных)
    └── нпс/                    ← папка для модульных (неканоничных) НПС
        └── [Имя]/
            └── [Имя].md        ← упрощённый лист (Шаблон Г из npcs_city.md)
```

**Правила именования папки:**
- Формат: `[месяц_ГГГГ]_[краткое_описание]` строчными буквами, через `_`
- Примеры: `ноябрь_2010_кошки_и_мышки`, `март_2010_la_defence`, `апрель_2010_фригольд`
- Латиница допустима только в названиях локаций (`la_defence`, `chatelet`)

**Правила именования главного файла модуля:**
- Имя файла = описательная часть папки без даты: `кошки_и_мышки.md`, `приглашение_домой.md`, `la_defence.md`
- Не использовать `модуль.md` — слишком универсально для навигации внутри папки

## 🔁 Протокол создания модуля

При добавлении нового модуля выполни шаги **в строгом порядке**:

> ⚡ Быстрый старт: `tools/new_module.ps1 "месяц_ГГГГ_название"` — создаёт структуру и задаёт вопросы интерактивно.

0. **Проверить связь с предыдущими хрониками.** Спросить: «Связан ли этот модуль с другими?» Если да — найти модули, у которых есть `финал.md`, показать список, дать выбрать. Ссылку на финал вынести в `сценарий.md` (раздел «Предпосылки»). Скрипт делает это автоматически.
1. **Создать папку:** `modules/[месяц_ГГГГ_описание]/` и подпапку `нпс/`.
2. **Создать главный файл `[описание].md`:** Заполнить таблицу параметров + краткое содержание. До сессии допустима заглушка.
3. **Создать `сценарий.md`:** Всегда. Структура: Предпосылки → Локации → НПС → Завязка → Сцены (3–5) → Кульминация → Варианты финала → Крючки → Парижский колорит. Заполняется Мастером до сессии или совместно с Claude по запросу.
4. **Создать `нпс.md`:** Всегда. Разделить на три секции: ПК / Каноничные НПС (ссылки на `characters/`) / Модульные НПС (ссылки на `нпс/`). Новых НПС определить как каноничных или модульных (см. ниже).
5. **Создать карточки новых локаций:** Если модуль использует локацию, для которой ещё нет карточки в `locations/` — создать её по правилам из `rules/portret.md`:
   - Парижская → `locations/[Район]_[Номер]/[Название_локации]/[Название].md`
   - Вне Парижа → `locations/Другие/[название_модуля]/[Название_локации]/[Название].md`
   - Карточка включает: атмосферу, сенсорную палитру, VtM-контекст, 3 крючка, промт + негативный промт.
6. **Создать `финал.md`:** Только если сессия разыграна и литературный финал написан.
7. **Добавить запись в `Stories_of_Paris.md`:** По протоколу из `rules/chronicle_paris.md` (шаги 1–7).

> 📋 **Полный операционный чеклист** (дневники участников, `open_threads.md`, валидация ссылок) — [rules/CHECKLIST.md](CHECKLIST.md) → секция 2.

## 🆕 Модульные (неканоничные) НПС

НПС, созданный специально для одного модуля, по умолчанию **неканоничный**: хранится в `modules/[название]/нпс/[Имя]/[Имя].md` по **Шаблону Г** из `rules/npcs_city.md` (упрощённый лист с характеристиками + промт изображения). В `characters/` не добавляется, в `characters_ALL.md` не вносится.

### Условия продвижения в каноничные

НПС становится каноничным при **выполнении всех трёх условий**:
1. Выжил (не уничтожен, не в торпоре постоянно).
2. Упомянут в `финал.md` как значимый персонаж.
3. Появился минимум в двух разных модулях — **или** пользователь явно запросил канонизацию.

**Протокол продвижения:**
1. Создать полную карточку по Шаблону А/Б/В в `characters/[линейка]/Имя/Имя.md`.
2. Добавить запись в `characters/characters_ALL.md`.
3. Добавить строку в `rules/npc_image_mapping.md`.
4. Обновить `нпс.md` в каждом модуле, где фигурирует НПС: заменить ссылку `нпс/Имя/Имя.md` на `../../characters/[линейка]/Имя/Имя.md`.
5. Удалить файл из `нпс/` (или оставить с пометкой `→ перемещён`).
6. Запустить `tools/validate_links.ps1`.

## ⚠️ Правила и ограничения

- Папка модуля создаётся **всегда** при добавлении записи в хронику — даже если модуль не разыгрывался как сессия (достаточно заглушки).
- Карточка локации создаётся **при первом появлении** локации в любом модуле. Если локация уже есть в `locations/` — создавать повторно не нужно, достаточно ссылки из `нпс.md` или основного файла модуля. Проверить наличие можно командой `tools/search.ps1 "Название локации"`.
- `финал.md` создаётся **только при необходимости** — не заранее.
- `нпс.md` создаётся **всегда для игровых сессий** — перечислять всех НПС модуля, включая ранее созданных в других модулях. **Исключение:** внесессионные события (`Тип: Внесессионное событие`) могут не иметь `нпс.md`, если НПС уже перечислены в тексте главного файла или не взаимодействуют активно с игроками.
- Пути в файлах модуля используют `../../` для выхода на уровень корня проекта.
- Ссылки на дневники: если событие до ноября 2010 → ссылаться на `Journal_[Имя]/retrospective.md`, не на месячный файл (`Journal_[Имя]/ГГГГ-ММ.md`).
- Если пользователь словесно описывает иную структуру → задать уточняющий вопрос с объяснением последствий (см. приоритетный принцип в `diary_rules.md`).
