## 2. Новый модуль

> Детальный протокол: [rules/module_rules.md](module_rules.md) | Хроника: [rules/chronicle_paris.md](chronicle.md)
> ⚡ Быстрый старт: `tools/new_module.ps1 "месяц_ГГГГ_название"` — создаёт структуру и задаёт вопросы интерактивно.

**До сессии:**
- [ ] *(Шаг 0)* Проверить связь с предыдущими хрониками — скрипт спросит автоматически
- [ ] Создать структуру: `modules/[месяц_ГГГГ_описание]/` + подпапка `нпс/`
- [ ] Создать `сценарий.md` — Предпосылки, Сцены (3–5), Кульминация, Финалы, Крючки, Парижский колорит
- [ ] Создать `нпс.md` с тремя секциями: ПК / Каноничные НПС / Модульные НПС
  - Каноничные НПС → ссылки на `characters/[линейка]/`; новые добавить по чеклисту **1. Новый персонаж**
  - Модульные НПС → карточки в `нпс/[Имя]/[Имя].md` по **Шаблону Г** из [rules/npcs_city.md](npcs_city.md)
- [ ] Проверить локации — если новая, создать карточку по [rules/portret.md](portret.md):
  - Парижская → `locations/[Район]_[Номер]/[Название]/[Название].md`
  - Вне Парижа → `locations/Другие/[название_модуля]/[Название]/[Название].md`
  - Проверить наличие: `tools/search.ps1 "Название локации"`

**После сессии:**
- [ ] Заполнить `[описание].md` (краткое содержание, ссылки на дневники)
- [ ] Создать `финал.md` — только если финал написан
- [ ] Добавить запись в `Stories_of_Paris.md` по протоколу из `rules/chronicle_paris.md` (шаги 1–7)
- [ ] Обновить `rules/open_threads.md` — новые нити из событий
- [ ] Создать/обновить дневники участников в `characters/[линейка]/[Персонаж]/Journal_[Имя]/[ГГГГ-ММ].md`
  - Правила: [rules/diary_rules.md](diary_rules.md)
- [ ] Проверить условия продвижения модульных НПС → `rules/module_rules.md`
- [ ] Запустить `tools/validate_links.ps1`

## 3. Переименование персонажа

> Самая дорогостоящая операция — затрагивает N файлов.

**Перед началом** — найти все вхождения старого имени:
```powershell
# Из корня проекта:
Select-String -Path (Get-ChildItem -Recurse -Filter "*.md").FullName -Pattern "СтароеИмя" | Select-Object Path, LineNumber, Line
```

**Чеклист:**
- [ ] Переименовать папку персонажа `characters/[линейка]/СтароеИмя/` → `characters/[линейка]/НовоеИмя/`
- [ ] Переименовать файл карточки внутри папки
- [ ] Обновить имя внутри переименованного файла карточки
- [ ] Обновить `characters/characters_ALL.md` — ссылка + упоминания
- [ ] Обновить `Stories_of_Paris.md` — все упоминания (имя в участниках, последствиях)
- [ ] Обновить `rules/chronicle_paris.md` — если имя есть в тексте протокола
- [ ] Обновить `rules/open_threads.md` — если имя фигурирует в нитях
- [ ] Обновить `rules/npc_image_mapping.md` — строка персонажа
- [ ] Обновить все карточки персонажей, ссылающихся на него (поле «Отношения», «Сир», «Дитя»)
- [ ] Обновить все файлы модулей (`modules/**/нпс.md`, основной файл модуля)
- [ ] Обновить заголовки дневников `characters/[линейка]/[Персонаж]/Journal_[Имя]/*.md`
- [ ] Запустить `tools/validate_links.ps1` → убедиться в 0 ошибках
