# ⚙️ СИСТЕМНОЕ ПРАВИЛО: Добавление и редактирование персонажей

> 📌 **Внимание!** Это правило применяется автоматически при любом запросе на создание, редактирование или добавление персонажа.

## 📂 Где хранятся данные

Данные по персонажам перенесены в папку `characters/` в корне проекта:

| Папка | Линейка WoD |
|---|---|
| `cities/<город>/characters/vampires` | Вампиры |
| `cities/<город>/characters/fairies` | Феи / Ченджлинги |
| `cities/<город>/characters/mortals` | Смертные |
| `cities/<город>/characters/werewolves` | Оборотни |
| `cities/<город>/characters/mages` | Маги |
| `cities/<город>/characters/hunters` | Охотники |

**Сводный справочник всех персонажей:** `cities/<город>/archive/characters_index.md`

---

## 🔁 Протокол добавления НПС

1. **Проверка дубликатов:** Поиск по имени, кличке, внешности, роли, сиру/дитя, домену. Если персонаж уже существует → редактируй существующую запись, не создавай новую.
2. **Выбор папки:** Определи линейку WoD и помести файл в соответствующую папку внутри `characters/`. Имя файла = имя персонажа (например: `<имя>.md`).
3. **Использование шаблона:** Вставляй карточку строго по структуре ниже.

## ⚙️ Правила

**Синхронизация связей:** Если указываешь сиру/дитя/союзника, проверь его карточку. Если есть → добавь обратную ссылку в поле «Отношения».

**Обработка пробелов:** Если данные отсутствуют, используй `[неизвестно]` или `⚠️ Требуется уточнение`. Не придумывай лор без явного запроса.

**Промт изображения:** После создания карточки обязательно:
1. Добавить `🎨 Промт для генерации изображения` в карточку (3 блока по структуре из `rules/portret.md`).
2. Добавить `## 🖼️ Изображения` в конец карточки (или `⏳ Изображение не предоставлено`).
3. Внести персонажа и цвет фона в таблицу `🎨 Схема фонов` в начале `rules/секции изображений карточки`.
4. Добавить краткую запись в `cities/<город>/archive/characters_index.md` в нужный раздел.

## ⚠️ Ограничения

- Не нарушай иерархию Камарильи/Клана/Двора без явного указания пользователя.
- Не смешивай термины WoD и Changeling без пояснений.
- Все имена, даты и топонимы должны соответствовать канону твоего сеттинга и его языку.
- Не включать «take pose from reference» в промт — поза задаётся при генерации.
- При работе с визуальными референсами сначала обновляй карточку персонажа, затем секции изображений карточки.
