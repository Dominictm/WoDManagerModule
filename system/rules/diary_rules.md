## 📂 ХРАНЕНИЕ — ЕДИНСТВЕННО ВЕРНАЯ СТРУКТУРА

> ⚠️ **Это приоритетное правило. Оно применяется всегда — даже если пользователь словесно описывает иную структуру. При расхождении — уточни, не импровизируй.**

```
characters/
└── [линейка]/                    ← vampires/ fairies/ mortals/ и т.д.
    └── [Имя персонажа]/          ← папка персонажа (там же карточка и изображения)
        ├── [Имя персонажа].md    ← карточка персонажа
        └── Journal_[Имя]/        ← папка дневников персонажа
            ├── [ГГГГ-ММ].md      ← события кампании (ноябрь 2010 и позже)
            └── retrospective.md  ← все события до ноября 2010 + флешбэки прошлых веков
```

**Правила именования:**
- Папка персонажа: точное имя из карточки персонажа (например, `Верене де Кюстин`, `Мишель Моро`)
- Папка дневников: `Journal_[Имя]`, где `[Имя]` — первое имя персонажа (например, `Journal_Верене`, `Journal_Тома`). Если два персонажа одной линейки имеют одинаковое первое имя — добавить фамилию: `Journal_Мишель_Альбер`, `Journal_Мишель_Моро`
- Файл периода: `ГГГГ-ММ.md` (например, `2010-11.md` для ноября 2010)
- Ретроспектива: всегда `retrospective.md`

### 📋 Когда создавать журнал

| Категория | Журнал | Комментарий |
|---|---|---|
| **Персонаж игрока (ПК)** | ✅ Обязательно | При создании ПК — дневники хранятся в `characters/[линейка]/Имя/` |
| **Ключевой НПС** (участвует в 2+ сессиях, у него есть активная сюжетная линия) | ✅ Рекомендуется | Создаётся при первой значимой сцене |
| **Эпизодический НПС** (одна сцена, нет внутренней жизни в нарративе) | ❌ Не создавать | Достаточно хроники и карточки |
| **Персонаж в торпоре** | ⏸ Не создавать до пробуждения | После выхода из торпора — `retrospective.md` задним числом, если нужно |

### ❌ ЗАПРЕЩЕНО

| Нельзя | Почему |
|---|---|
| Создавать файлы дневников напрямую в папке персонажа (без `Journal_[Имя]/`) | Дневники хранятся в подпапке `Journal_[Имя]/` внутри папки персонажа |
| Называть папку дневников иначе: `journals/`, `дневники/`, `diary/` | Единственный допустимый формат: `Journal_[Имя]` |
| Называть файлы по событию: `ноябрь_2010_кошки_и_мышки.md` | Только формат `ГГГГ-ММ.md` |
| Создавать мастер-файл типа `дневник_Ямалы.md` | Коллекторов нет — только периодные файлы |

## 🔁 Протокол зеркального отражения

При **добавлении**, **изменении** или **удалении** события в `Stories_of_Paris.md` выполни шаги **в строгом порядке**:

1. **Идентификация участников:** Извлеки из записи хроники всех присутствующих/затронутых НПС и ПК.
2. **Проверка существующих дневников:** Если у участника уже есть файл `Journal_[Имя]/ГГГГ-ММ.md` за этот месяц → добавь новую сцену в конец файла через разделитель `---`. Если файла нет → создай его (папку `Journal_[Имя]/` создать заранее, если отсутствует). Один файл = один месяц, сколько бы сцен в нём ни было.
3. **Генерация перспективы:** Для каждого участника создай запись, строго соответствующую его профилю из `characters/[линейка]/Имя.md` (клан, секта, деранжементы, отношения, тон восприятия).
4. **Кросс-ссылка:** В каждой записи добавь блок `🔗 Зеркальная ссылка`.
5. **Валидация фактов:** Все дневники описывают **один исход** из хроники. Различия допустимы только в интерпретации, эмоциях, скрытых деталях или сознательном искажении.
6. **Подтверждение:** Верни отчёт:

```
✅ Дневники синхронизированы: [Дата/Название события]
👥 Обновлены записи: [Имя1, Имя2, Имя3...]
🔗 Кросс-ссылки установлены: Да
📝 Готово к литературной генерации: Да
```

### ⚠️ Правила обработки изменений

- Если факт изменён ретроспективно → обнови **все** связанные дневники. Старые версии помечай `🔄 РЕДАКТИРОВАНО: [причина]`.
- Если участник не мог знать о событии (физически отсутствовал, в торпоре) → его дневник либо не создаётся, либо содержит `📝 Косвенная реакция: [реакция на последствия/сплетни]`.
- Противоречия с хроникой допустимы **только** как признак безумия, лжи или искажённого восприятия. Всегда помечай: `⚠️ ИСКАЖЕНИЕ ВОСПРИЯТИЯ`.

## 📜 Общие требования к прозе

- **Строгий POV:** Первое лицо или глубокое третье лицо ограниченное. Никакого всеведения.
- **Маскарад:** Прямые упоминания вампирской природы/дисциплин — только через метафоры, шёпот, внутренние ощущения или закодированные записи.
- **Атмосфера Парижа:** Погода, архитектура, звуки (Сена, метро, колокола, сирены), запахи (старые книги, кровь, озон, дождь) — как эмоциональный фон.
- **Длина:** 150–400 слов на запись, если не запрошено иное.
- **Даты:** Могут быть точными или поэтичными («ночь, когда Сена пахла ржавчиной», «после третьего звона Нотр-Дама»), в зависимости от характера автора.

## 🔄 Механизм обновления при изменении хроники

При команде `🔄 Обновить дневники: [Дата/Название]`:

1. Сравни новую версию события в `Stories_of_Paris.md` с предыдущей.
2. Выяви изменённые факты: участники, исход, локации, политические последствия.
3. Для каждого затронутого персонажа:
   - Факт изменился → отредактируй соответствующий абзац в дневнике.
   - Добавлен новый участник → создай для него запись.
   - Участник исключён → пометь `🚫 УДАЛЕНО ИЗ КОНТЕКСТА: [причина]` или удали.
4. Перестрой `🔗 Зеркальные ссылки` во всех затронутых записях.
5. Верни отчёт с указанием, какие строки изменены и почему.

## ⚠️ Ограничения и валидация

- Не создавай дневниковые записи без привязки к событию в `Stories_of_Paris.md`.
- Одна запись = один разум. Не смешивай перспективы в одной записи.
- Все искажения фактов помечать `⚠️ ИСКАЖЕНИЕ ВОСПРИЯТИЯ` или `🎭 РОЛЕВАЯ МАСКА`.
- При генерации всегда проверяй карточку персонажа в `characters/[линейка]/[Имя]/[Имя].md` на деранжементы, отношения и клановые особенности.
- При конфликте между дневником и хроникой → хроника приоритетна. Дневник корректируется или помечается как ненадёжный источник.

## 📅 Граница ретроспективы

> Все события **до ноября 2010** записываются в `Journal_[Имя]/retrospective.md` вне зависимости от персонажа. Месячные файлы `Journal_[Имя]/ГГГГ-ММ.md` создаются **начиная с ноября 2010**.  
> `retrospective.md` — **дополнение** к месячным файлам, не замена. Оба типа файлов существуют параллельно.
