# Автотесты VTM Chronicle Manager

Два набора, оба сами поднимают сервер на отдельном порту и **убирают за собой**
одноразовый тестовый город (ничего из реальных данных не трогают).

## Запуск

```bash
cd tests
npm install            # один раз (ставит selenium-webdriver; ChromeDriver скачает Selenium Manager)

npm run e2e            # End-to-End: API + файловая система (без браузера, ~3 c)
npm run ui             # UI: Selenium, видимый Chrome
npm run ui:headless    # UI без окна (для CI)
npm test               # оба набора подряд
```

Или из проводника: **`tests\run.bat`** (запускает оба, видимый браузер).

Отчёты: `tests/report_e2e.html`, `tests/report_ui.html` (тёмная тема, перезаписываются).

## Что покрыто

**`e2e.js` (17 проверок)** — сквозной цикл Рассказчика:
`new_city` → `new_npc` → `new_location` → `/api/log-session` (предпросмотр →
проверка отклонения устаревшего хэша 409 → запись, включая **инлайн-создание НПС**)
→ `/api/status` → `/api/chronicles` → `close_chronicle` (статус + закрытие нитей)
→ `build_city_events` → линтер `validate_cards --strict`.

**`ui_selenium.js` (21 проверка)** — реальный фронтенд в Chrome:
загрузка SPA и редирект на `?city=`, навигация по всем 8 разделам, карточки панели,
грид персонажей + поиск, переключатель города, вкладка «Инструменты» — создание
города/НПС/локации через UI (`/api/tool`), «Пересобрать индекс», «Проверить ссылки».

## Конфигурация (env)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `HEADLESS=1` | UI без окна браузера | выкл (видимый) |
| `E2E_PORT` / `UI_PORT` | порт тестового сервера | 3097 / 3098 |
| `VERBOSE=1` | печатать stdout/stderr сервера | выкл |

Требование UI-тестов: установлен Google Chrome.

## Устаревшие файлы

`ui_tests.js`, `run_tests.js`, `run_all_tests.js`, `report_all.html` и их `.bat`
написаны **до** перехода на структуру `cities/` и больше не соответствуют проекту
(обращаются к `localhost:3000` без `?city=`, к удалённым PS-инструментам и плоской
структуре). Их можно удалить — актуальны `e2e.js` и `ui_selenium.js`.
